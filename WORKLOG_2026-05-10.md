# 2026-05-10 开发日志

## 1.11.16 · 新增账户门店修复与演示版

- 复查 AQC-N 管理员设置新增账户链路、后端 `AdminUserCreateRequest` / `AdminUserUpdateRequest` schema，以及 account-symuse 的用户创建、更新、AQC 用户组 upsert 接口。
- 定位 HTTP 422 根因：请求体中的 `shopIds` 有机会从历史单选控件或旧状态传成单个数字，例如 `27`，而 AQC-N 后端要求 `list[int]`，Pydantic 在进入路由前直接拒绝。
- 在前端 `AccountAdminView.vue` 提交 payload 前将 `shopIds` 统一收敛为正整数数组，同时保留 `shopId` 作为第一个门店兼容字段。
- 在后端 `schemas.py` 为新增 / 编辑账户请求增加 `shopIds` 前置校验，兼容单个数字、字符串、元组、集合、空值和数组，避免同类旧客户端请求再次触发 422。
- 新增前端 demo 模式开关：`demo.aqc.symuse.com` 自动进入，也可本地用 `?demo=1` 开启，登录态使用演示 token，不写入真实登录存储。
- 新增浏览器内 demo API 数据层，固定返回演示账户、员工、门店、商品图片、销售记录、订单、工单、通知和报告基础数据。
- 演示模式下 `POST /sales/records` 和 `POST /work-orders` 仅更新当前前端会话内存或返回模拟成功，不请求服务器，不改变真实商品、库存、销售数据。
- 将现有本地 mock 的全局搜索、销售录入、销售记录和工单管理扩展为 demo 域名自动启用，并沿用现有深色模式主题变量。
